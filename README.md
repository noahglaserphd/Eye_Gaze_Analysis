# Video-Based Gaze Analysis from Eye-Tracking Overlays

An open-source research pipeline for extracting and analyzing gaze behavior from **video recordings that contain a visible gaze overlay**.

The system detects gaze markers embedded in recorded videos and converts them into structured gaze data such as gaze coordinates, fixations, and saccades. These data can then be analyzed through statistical summaries, spatial heatmaps, scanpaths, and an interactive dashboard.

This pipeline was developed primarily for **human–computer interaction (HCI), usability testing, and gameplay research**, but it can be used in any situation where gaze overlays appear in recorded video.

Unlike traditional eye-tracking analysis tools that depend on proprietary ecosystems, this approach works directly with **screen recordings containing a gaze cursor or overlay**, allowing analysis without access to vendor APIs or specialized analysis software.

---

# Project Workflow

The project has two stages:

1. **Offline Processing Scripts**  
   Extract gaze data from videos and compute gaze metrics.

2. **Interactive Dashboard**  
   Explore sessions, inspect time windows, view visualizations, and export results.

The dashboard **does not process raw videos**. It visualizes data that has already been generated by the processing scripts.

# Step 1 — Prepare Videos

Place recordings containing gaze overlays inside the `videos/` folder.

Example:

```
videos/
   Rec1.mkv
   Rec2.mkv
   Rec3.mp4
```

Each filename becomes the **session identifier** used throughout the analysis.

---

# Step 2 — Extract Gaze Coordinates From Video

Run the gaze extraction script:

```
python extract_dot.py
```

This script scans all videos inside the `videos/` directory and detects the gaze overlay position in each frame.

The result is a time series of gaze coordinates.

---

# Step 3 — Process Gaze Data

Convert extracted gaze coordinates into structured gaze data:

```
python analyze_gaze_csv.py
```

This step produces:

- gaze samples
- fixation events
- saccade events

---

# Step 4 — Generate Fixation Statistics

```
python summarize_fixations.py
```

This script computes summary statistics describing fixation behavior.

---

# Step 5 — Generate Session Visualizations

```
python make_figures.py
```

This produces visualizations such as:

- fixation heatmaps
- scanpath diagrams
- fixation duration distributions
- saccade amplitude distributions

---

# Step 6 — Generate Time Window Metrics

```
python window_analysis.py
```

This divides each session into **fixed 30-second windows** and calculates gaze metrics for each window.

These windowed metrics allow researchers to analyze **how visual attention changes over time**.

---

# Step 7 — Generate Aggregate Figures

```
python make_aggregate_figure.py
```

This script produces summary figures across all sessions.

---

# Step 8 — Launch the Interactive Dashboard

Start the dashboard:

```
streamlit run app.py
```

The dashboard provides interactive exploration of gaze behavior.

---

# Dashboard Capabilities

The dashboard allows researchers to:

- select recorded sessions
- inspect **30-second time windows**
- view fixation heatmaps
- visualize scanpaths
- examine fixation duration distributions
- analyze saccade amplitude distributions
- view synchronized video clips
- export charts and video segments
- export full-session outputs

The dashboard is designed for **exploratory analysis and qualitative inspection** of gaze behavior.

---

# Time Window Analysis

Each session is divided into **30-second time windows**.

Example segmentation:

```
0–30 seconds
30–60 seconds
60–90 seconds
```

For each window the system calculates:

- fixation count
- mean fixation duration
- total fixation time
- mean saccade amplitude

These metrics help identify **temporal shifts in attention or task difficulty**.
